检测

既然我们已经实现了跳跃，但此时我们可以发现——居然可以无限跳，这肯定是不合理的，所有就想到只能在地面上跳跃，由此引出我们的检测是否位于地面。

检测我们另起一个脚本，名为 PhysicsCheck.cs 以后无论是地面检测还是蹲下亦或是悬挂，都是将逻辑写在这个脚本当中。要检测是否在地面，我们使用一下这个 API


后面两个参数我们暂时不用考虑，只关注前三个参数：

  • point：检测的中心点
  • radius：检测的圆的半径
  • layerMask：检测的图层

PS：此处略微说一下 Layer 和物体组件 Sprite Renderer 中的 Sorting Layer 的区别：

  • Layer：层级。用于物体的逻辑分层，主要用于这种检测
  • Sorting Layer：排序层级。影响物体渲染的顺序。排序越靠后的物体越在上面，统一层级中 Order in Layer 数字越大排序越前

有了这个接口以后，我们就能以圆形的方式检测是否在地面这个图层中了。（需将地面的 Tilemap 提前加入 Ground 这个图层）

关于一些 Player 的注意事项

Collider 的选择

关于 Player 和 NPC，最好选择 Capsule Collider 2D，也就是胶囊形状的碰撞体

人物一直有个 x 轴的力导致可以贴在墙上怎么办？

右键新建一个 Physics Material 2D，将其 Firction 也就是摩擦力改为 0 即可


接着将其加入 Collider 组件的 Material 即可

人物高速落下卡进地面又瞬间弹出怎么办?

我们只需去到 Player 的 Rigidbody 2D 里面，将 Collision Detection 改为 Continuous 即可，也就是将碰撞检测模式改为连续检测。同时我们也将 Interpolate 属性改为 Interpolate，采用插值的形式使我们的运动更流畅，Interpolate 运动插值基于前几帧的位置。

Run

接下来是制作跑步的状态机逻辑，要判断是否跑步，我们只需判断其 x 方向的 velocity 即可。首先在 Animator 的左侧创建一个 float 类型的参数 velocityX。接着在状态机中连线并且添加转换状态的条件。



我们设定 Idel -> Run 的条件为 velocityX 大于 0.1 的情况，不直接大于 0 是为了避免一些精度问题。同样的，Run -> Idle 肯定是小于 0.1 的时候。此处还有两个值得注意的选项：

  • Has Exit Time：可以理解为是否有一个结束的时间，当满足动画切换条件后并不马上切换动画,而是在前一个动画播放到退出时间后才切换，此处的 Exit Time 时相对转换前的事件而言的，此处是不需要勾选的
  • Fixed Duration：勾选之后, Transition Duration 的单位将会变成秒,否则是相对转换前动画的时间的相应倍数，这里也不需要勾选

Blend Tree 与 跳跃

我们可以看出，跳跃的动画分为三部分：Jump -> JumpToFall -> Fall。如果直接使用状态机将每一个连线连起来，那判断条件和维护过程都会非常艰难。因此我们使用 Blend Tree 也就是混合树，将三个动作整合成一个状态。

设置一个新的 float 变量 velocityY 来作为判断条件，在右边我们将三个动画添加，并且设置 velocityY 的值变为多少时就切换动画。此时我们就得到了一个线性变化特别流程且好维护的状态。
